1. 概要
================================================================================

本ガイドラインは、エスペラントの文章をすべて語根分解し、2文字以上の各語根に
日本語のルビ注釈を付したHTML文書を、生成AIに修正させる際の参考資料である。

修正作業の主なカテゴリ：
  - rt注釈テキストの語義の誤り修正
  - ルビ未付与の単語への注釈追加
  - CSSクラスの適正化（ピクセル幅比率ベース）
  - 同一語根に対する注釈の統一
  - 改行位置（<br>）の整合
  - HTML構造上の問題の修正

【必要ファイル（同一ディレクトリに配置すること）】
  (1) Unicode_BMP全范围文字幅(宽)_Arial16.json
      → 各Unicode文字の Arial 16px におけるピクセル幅データ
      → CSSクラス決定に必須（§2.3 で詳述）

2. CSS設計の理解（★最重要）

────────────────────────────────────────────────────────────────────────────────
2.1 CSSクラス一覧（8段階）
────────────────────────────────────────────────────────────────────────────────

本文書で使用するルビのCSSクラスは以下の8段階である。修正作業の前に、
この体系を正確に理解しておく必要がある。

  ┌──────────┬──────────┬─────────────┬──────────────────────┐
  │ クラス名  │フォント  │ margin-top  │ 用途                  │
  │           │ サイズ   │             │                       │
  ├──────────┼──────────┼─────────────┼──────────────────────┤
  │ XXXS_S   │ 0.3em    │ -8.3em      │ 3行注釈（<br>×2）     │
  │ XXS_S    │ 0.3em    │ -7.2em      │ 2行注釈（<br>×1）     │
  │ XS_S     │ 0.3em    │ -6.1em      │ 1行・極小フォント     │
  │ S_S      │ 0.4em    │ -4.85em     │ 1行・小フォント       │
  │ M_M      │ 0.5em    │ -4.00em     │ 1行・中フォント       │
  │ L_L      │ 0.6em    │ -3.55em     │ 1行・やや大フォント   │
  │ XL_L     │ 0.7em    │ -3.20em     │ 1行・大フォント       │
  │ XXL_L    │ 0.8em    │ -2.80em     │ 1行・最大フォント     │
  └──────────┴──────────┴─────────────┴──────────────────────┘

────────────────────────────────────────────────────────────────────────────────
2.2 基本ルール
────────────────────────────────────────────────────────────────────────────────

【<br>個数とクラスの対応（絶対厳守）】
  (a) <br>が2つ含まれる注釈（3行） → 必ず XXXS_S を使用
  (b) <br>が1つ含まれる注釈（2行） → 必ず XXS_S を使用
  (c) <br>を含まない注釈（1行）    → XS_S～XXL_L のいずれかを使用
  (d) XXXS_SやXXS_Sを1行注釈に使うと、margin-topが過大となり
      ルビが本文テキストの上方に浮いてしまうため、絶対に避ける。

【★★★ 既存CSSクラスの変更禁止（最重要ルール）★★★】
  (e) 既にファイル内に存在するrubyタグのCSSクラスは、絶対に変更しない。
      元のCSSクラスはピクセル幅計算（§2.3）によって正確に決定された
      ものであり、AIが文字数等の近似で再計算しても正確な値にはならない。
  (f) 新規にrubyを追加する場合、同一の (rb, rt) ペアがファイル内に
      既に存在すれば、その既存クラスをそのまま使用する。
  (g) ファイル内に前例のない全く新規の (rb, rt) ペアに限り、
      §2.3 のピクセル幅アルゴリズムでCSSクラスを決定する。

【同一ペアの統一ルール】
  (h) 同一の (rb, rt) ペアには同一のCSSクラスを割り当てる。
      同じ語根＋注釈でクラスがバラバラだと表示が不統一になる。

────────────────────────────────────────────────────────────────────────────────
2.3 CSSクラス決定アルゴリズム（ピクセル幅ベース）
────────────────────────────────────────────────────────────────────────────────

CSSクラスは「rt側のピクセル幅 ÷ rb側のピクセル幅」の比率（ratio）で
決定される。文字数の比率ではない。

※ このアルゴリズムは esp_replacement_json_make_module.py 内の
   output_format 関数に実装されているものと同一である。

■ ピクセル幅の計算方法

  Unicode_BMP全范围文字幅(宽)_Arial16.json を読み込み、
  文字列の各文字のピクセル幅を合計する。

  計算式:
    pixel_width(text) = Σ char_widths_dict.get(ch, 8)  （各文字 ch について）

  ※ JSONに存在しない文字はデフォルト幅 8 として扱う。

  参考値（Arial 16px、JSONからの実測値）:
    ラテン文字: a≈8.9, i≈3.6, m≈13.3, w≈11.6（文字により幅が大きく異なる）
    漢字: 概ね 16.0
    カタカナ: 概ね 16.0
    エスペラント字上符: ĉ≈8.0, ĝ≈8.9, ĥ≈8.9, ĵ≈3.6, ŝ≈8.0, ŭ≈8.9

■ 比率の計算

  ratio = pixel_width(rt文字列) ÷ pixel_width(rb文字列)

  ※ rt文字列: <br>タグを除去した純テキスト部分で計算する。
     例: 「継続<br>行為」→ 「継続行為」のピクセル幅で計算。

■ 閾値テーブル（ratio から CSSクラスへの変換）

  ┌─────────────────┬──────────┬──────────────────────────────────────┐
  │ 条件             │ クラス    │ 備考                                │
  ├─────────────────┼──────────┼──────────────────────────────────────┤
  │ ratio > 6.0     │ XXXS_S   │ <br>を2つ挿入（3行化）              │
  │ ratio > 3.0     │ XXS_S    │ <br>を1つ挿入（2行化）              │
  │ ratio > 2.25    │ XS_S     │ 1行                                 │
  │ ratio > 1.8     │ S_S      │ 1行                                 │
  │ ratio > 1.5     │ M_M      │ 1行                                 │
  │ ratio > 9/7     │ L_L      │ 1行（9/7 ≈ 1.2857）                │
  │ ratio > 1.125   │ XL_L     │ 1行（1.125 = 9/8）                 │
  │ それ以外         │ XXL_L    │ 1行                                 │
  └─────────────────┴──────────┴──────────────────────────────────────┘

  ※ 閾値は上から順に判定する（最初に該当した条件で確定）。
  ※ 閾値の正確な値:
     6（独立値）, 9/3(=3), 9/4(=2.25), 9/5(=1.8), 9/6(=1.5),
     9/7(≈1.28571), 9/8(=1.125)

■ <br>の挿入位置

  XXXS_S の場合（<br>×2）:
    rt文字列の合計ピクセル幅を3等分し、
    1/3地点と2/3地点に達した直後の文字境界に<br>を挿入する。
    （insert_br_at_third_width 相当）

  XXS_S の場合（<br>×1）:
    rt文字列の合計ピクセル幅を2等分し、
    1/2地点に達した直後の文字境界に<br>を挿入する。
    （insert_br_at_half_width 相当）

  ※ 挿入位置もピクセル幅ベースであり、文字数の等分ではない。

────────────────────────────────────────────────────────────────────────────────
2.4 AIによる新規rubyのCSS決定手順
────────────────────────────────────────────────────────────────────────────────

新たにrubyタグを追加する場合は、以下の手順に従う。

  Step 1: 同一 (rb, rt) ペアがファイル内に既存か検索する
          → 存在すれば、そのクラスをそのまま使用する（以降のStepは不要）

  Step 2: Unicode_BMP全范围文字幅(宽)_Arial16.json のピクセル幅データを
          使い、rb側・rt側それぞれの合計ピクセル幅を計算する

  Step 3: ratio = rt側ピクセル幅 ÷ rb側ピクセル幅 を求める

  Step 4: §2.3 の閾値テーブルに従ってクラスを決定する

  Step 5: ratio > 3.0 の場合はrt内に <br> を挿入する
          - ratio > 6.0 → <br>×2（ピクセル幅の1/3・2/3地点）
          - ratio > 3.0 → <br>×1（ピクセル幅の1/2地点）

  ★ Step 1 が最優先。既存ペアのクラスに合わせることが最も重要。
  ★ JSONファイルを読めない場合は、Step 1 のみで対応する。
     それでも前例のない新規ペアは、CSSクラスを空欄にしてユーザーに
     確認を求めるか、文字数比率による粗い近似値を使うが、
     「確定値ではなく近似値である」ことを必ず明記すること。

3. 修正カテゴリ一覧

────────────────────────────────────────────────────────────────────────────────
3.1 注釈テキストの誤り（語義の間違い）
────────────────────────────────────────────────────────────────────────────────

最も重大なカテゴリ。エスペラント語根の意味を誤って日本語訳している箇所。

【典型的な誤りパターン】

(A) 類似前置詞の取り違え
    - pro（〜ゆえに）を「ために」と誤訳 ← 「ために」は por の意味
    - 修正前：<ruby>Pro<rt class="S_S">ために</rt></ruby>
    - 修正後：<ruby>Pro<rt class="S_S">〜ゆえに</rt></ruby>
    → por（目的）と pro（理由）の違いを正確に区別すること

(B) 語根の基本義の取り違え
    - paŝ（一歩）を「歩く」と誤訳 ← 「歩く」は marŝi の意味
    - 修正前：<ruby>paŝ<rt class="XL_L">歩く</rt></ruby>
    - 修正後：<ruby>paŝ<rt class="XL_L">一歩</rt></ruby>

(C) 接尾辞の基本義の揺れ
    - eg（強大）を「巨大」「非常に」と表記揺れ
    - an（成員）を「会員」と表記
    → 各接辞は1つの統一された訳語を使用すること

(D) 英語のまま残された注釈
    - dev（義務）が "must" のまま
    → 日本語注釈に統一すること

(E) HTMLタグの混入
    - neniam（決して〜ない）の注釈に「td>」が混入
    - 修正前：決して~ないtd>
    - 修正後：決して~ない
    → 注釈テキスト内にHTMLタグ残骸がないか検査すること

【よく見られる語義の誤り例】

  語根         誤            正              備考
  ─────────────────────────────────────────────────────
  pro/Pro      ために        〜ゆえに        por との混同
  paŝ          歩く          一歩            marŝi との混同
  tro          見える        過度に          完全な誤り
  rezult       起きる        結果            動詞形の誤訳
  dev          must          義務            英語が残存
  sin/Sin      懐            自身を          再帰代名詞の誤訳
  civit        自治都市      都市(国家)      語根はcivitas由来。civitano=市民は-an-で表現
  ŝarĝ        積荷          (に)荷を負わす  品詞方向の修正

────────────────────────────────────────────────────────────────────────────────
3.2 ルビ未付与の単語（注釈の欠落）
────────────────────────────────────────────────────────────────────────────────

ルビなしの裸テキストとして残されたエスペラント単語を検出して注釈を付与する。

【ルビが欠落しやすい単語カテゴリ】

(A) 定冠詞 La / la
    - 文頭の "La" が裸テキストのまま放置されるケースが非常に多い
    → 全文検索で La/la の未注釈箇所を洗い出すこと

(B) 固有名詞（地名・人名・時代名）
    - Japanio（日本）, Usono（アメリカ）, Meiĵi（明治）, Edo（江戸）,
      Ŝooŭa（昭和）, Tokio（東京）, Oosako（大阪）, Nagasaki（長崎）等
    → 固有名詞でも語根分解可能なものはルビを付与する
    → [地名]、[人名] などの分野タグを注釈に含める

(C) 国名・民族名の形容詞
    - ĉina（中国の）, japana（日本の）, nederlanda（オランダの）等
    → 国名系語根は、対応する日本語の国名をルビとする

(D) 一般語彙の漏れ
    - agrikulturo（農業）, erao（時代）, restaŭro（維新）等
    - eĉ（さえ）が大量に裸テキスト
    → 2文字以上のエスペラント語根はすべて注釈対象

(E) 複合語の一部
    - komputilo の "komput" 部分が裸テキストで "il" だけにルビ
    → 複合語であっても各語根に個別のルビを付与すること

────────────────────────────────────────────────────────────────────────────────
3.3 CSSサイズクラスの不適切な割当て
────────────────────────────────────────────────────────────────────────────────

ルビの表示位置やサイズが不適切になるCSSクラスの選択ミス。

【チェックポイント】

(A) 3行注釈（<br>×2）に XXS_S を使用 → XXXS_S に修正
    - 例：「(に)荷<br>を負<br>わす」に XXS_S → XXXS_S
    - 例：「~すれ<br>ばする<br>ほど」に XXS_S → XXXS_S
    → <br>の個数を数え、適切なクラスを確認すること

(B) 1行注釈に XXS_S を使用 → XS_S等に修正
    → 1行注釈に XXS_S/XXXS_S が使われていないか全件検査

(C) 同一注釈テキストの同一語根でクラスが不統一
    → 同一の語根＋注釈の組み合わせでクラスが統一されているか確認

(D) 新規追加rubyのクラスが不適切
    → ピクセル幅比率ベースで決定されているか確認（§2.3参照）
    → ★ただし、既存rubyのクラスは絶対に変更しないこと（§2.2(e)参照）

────────────────────────────────────────────────────────────────────────────────
3.4 同一語根に対する注釈の不統一
────────────────────────────────────────────────────────────────────────────────

同じ語根に異なる注釈テキストが使われている不整合。
学習者の混乱を防ぐため、可能な限り統一する。

【統一すべきケース（表記揺れ）】

  語根         揺れの例              統一先
  ─────────────────────────────────────────────────────
  eg           強大 / 巨大 / 非常に  → 強大
  eĉ           (~)さえ / さえも      → (~)さえ
  apart        別々の / 離れて       → 別々の
  san          健康な / 健康         → 健康な
  ĝeneral      全般の / 一般的な     → 全般の
  urĝ          さし迫る / 緊急に     → さし迫る
  aprior       先験的な / 先験的に   → 先験的な
  aposterior   後験的な / 後験的に   → 後験的な
  do/Do        それゆえ / そこで     → それゆえ
  mekanism     機構 / メカニズム     → 機構
  Am/am        愛する / 愛           → 愛する
  demokrat     民主 / 民主主義者     → 民主
  anim         魂 / 心               → 魂

【統一すべきでないケース（正当な多義語）】

  同形異義語は文脈に応じた異なる注釈が正しい。

  語根    意味1          意味2              備考
  ──────────────────────────────────────────────────
  en      中で           円                 前置詞 vs 通貨名
  kun     (~と)共に      [日]訓読み         前置詞 vs 日本語概念
  ĉar     なぜなら       荷車               接続詞 vs 名詞語根
  radio   ラジオ         放射線             異なる語義
  sek     乾いた         [単]秒             形容詞 vs 単位
  kultur  文化           (を)栽培する       抽象義 vs 農業義
  kod     暗号           コード             暗号義 vs 情報学義
  ĉiel    空             色々に             名詞語根 vs 相関詞副詞

  → 多義語の判断にはエスペラントの語源知識が必要

────────────────────────────────────────────────────────────────────────────────
3.5 改行位置（<br>）の不整合
────────────────────────────────────────────────────────────────────────────────

2行注釈における <br> の挿入位置が不統一なケース。

  語根    不統一例                   統一先
  ──────────────────────────────────────────
  ĵus     たっ<br>た今 / たった<br>今 → たった<br>今
  ia      ある種<br>の / ある<br>種の  → ある<br>種の

  → 同一注釈テキストで <br> 位置が異なる箇所を検索し統一すること
  → <br>の挿入位置はピクセル幅ベースの等分点が原則（§2.3参照）

────────────────────────────────────────────────────────────────────────────────
3.6 HTML構造上の問題
────────────────────────────────────────────────────────────────────────────────

(A) rubyタグとrtタグのバランス
    → <ruby>の開始タグ数 = </ruby>の閉じタグ数 を確認
    → <rt の開始数 = </rt> の閉じ数 を確認

(B) rtタグ内へのHTMLタグ残骸の混入
    → 「td>」等が注釈テキストに紛れ込んでいないか検査

(C) 文書全体のコード整形
    → 1行に大量のrubyを詰め込まず、適度に改行する（目安: 5個以下/行）

================================================================================
4. エスペラント語学上の注意点
================================================================================

────────────────────────────────────────────────────────────────────────────────
4.1 接頭辞を含む外来語の誤分解
────────────────────────────────────────────────────────────────────────────────

エスペラントの接頭辞（mal-, re-, ek-, dis-, el-, inter-, pri- 等）と
同じ文字列で始まる外来語を誤って分解しないこと。

  【正しく1語根として扱うべき単語の例】
  element   → el + ement × （elは接頭辞ではない）
  interes   → inter + es × （interは接頭辞ではない）
  princip   → pri + ncip ×
  preciz    → pre + ciz ×
  prezent   → pre + zent ×
  profesi   → pro + fesi ×
  program   → pro + gram ×
  surpriz   → sur + priz ×
  kondiĉ    → kon + diĉ × （konはEの接頭辞ではない）
  konstru   → kon + stru ×
  diskriminaci → dis + kriminaci × （1語根の外来語）

  → 外来語（国際語彙）はエスペラント接頭辞で分解しない

────────────────────────────────────────────────────────────────────────────────
4.2 同形異義語（多義語）の扱い
────────────────────────────────────────────────────────────────────────────────

同じ綴りで異なる意味を持つ語根がある。これらは文脈に応じて
異なる注釈を付けるのが正しい。

  ĉar → 接続詞「なぜなら」 vs 名詞語根「荷車」（ĉaro）
  en  → 前置詞「中で」 vs 名詞「円」（日本の通貨）
  dis → 接頭辞「分散」 vs 他の用法
  pas  → 動詞語根「通過する」（最も一般的な意味）
  pasi → 名詞語根「情熱」（pasio） vs 語源的要素「全」（pasigrafio内）

  → 文脈から語義を判断し、適切な注釈を選択する
  → 同形異義語は「統一すべき表記揺れ」と区別する
  → セミコロン等で訳語を無理に詰め込まず、出現ごとに適切な訳語を選ぶ

────────────────────────────────────────────────────────────────────────────────
4.3 国名・民族名の注釈方針
────────────────────────────────────────────────────────────────────────────────

国名語根の注釈には「〜人」を付けない。

  修正前        修正後       理由
  ─────────────────────────────────────
  イギリス人 → イギリス    語根はangl-（イギリスの）であり
  フランス人 → フランス    「人」はエスペラントの-anoで表現
  ドイツ人   → ドイツ      語根の基本義は国/地域名
  ロシア人   → ロシア
  スペイン人 → スペイン
  ポーランド人→ポーランド
  ヘブライ人 → ヘブライ

  → 語根の注釈は語根の基本義（国名）とし、
    「〜人」は接尾辞 -an- の注釈で表現する

────────────────────────────────────────────────────────────────────────────────
4.4 語根の品詞的性質と注釈形式
────────────────────────────────────────────────────────────────────────────────

エスペラントでは語根に文法語尾（-o, -a, -e, -i, -as等）を付けて
品詞を変える。ルビ注釈は語根の基本義を1つの形式で統一する。

  原則：同一語根には文脈によらず同一注釈テキストを使用する

  例：san → 常に「健康な」（「健康」「健康に」等と揺らさない）
      apart → 常に「別々の」（「離れて」等と揺らさない）
      aprior → 常に「先験的な」（「先験的に」と揺らさない）

  理由：文法語尾（-a, -o, -e等）がそれぞれ「形容詞」「名詞」「副詞」
        という品詞情報を持っているため、語根側の注釈は基本形で統一する

5. 修正作業の具体的手順
================================================================================

以下の手順で、効率的かつ網羅的に修正を進めることを推奨する。

────────────────────────────────────────────────────────────────────────────────
第1段階：全ルビペアの抽出と一貫性検査（最優先）
────────────────────────────────────────────────────────────────────────────────

  (1) ファイルからすべての <ruby>語根<rt class="X">注釈</rt> ペアを抽出
  (2) 同一語根に対して異なる注釈がある箇所をリストアップ
  (3) 各ケースを「表記揺れ（統一すべき）」か「多義語（維持すべき）」に分類
  (4) 表記揺れの場合、多数派の注釈を正とし、少数派を修正

────────────────────────────────────────────────────────────────────────────────
第2段階：CSSクラスの適切性検査
────────────────────────────────────────────────────────────────────────────────

  (1) <br>を2つ含むrtにXXXS_S以外が使われていないか検査
  (2) <br>を1つ含むrtにXXS_S以外が使われていないか検査
  (3) <br>を含まないrtにXXXS_SまたはXXS_Sが使われていないか検査
  (4) 同一語根＋注釈でクラスが不統一でないか検査
  ★ 既存クラスの値自体を変更しようとしないこと（§2.2(e)参照）

────────────────────────────────────────────────────────────────────────────────
第3段階：ルビ未付与語の検出
────────────────────────────────────────────────────────────────────────────────

  (1) </ruby>タグと<ruby>タグの間に3文字以上のアルファベット列がないか検査
  (2) 特に定冠詞 La/la の未注釈箇所を重点的に検査
  (3) 固有名詞（地名・人名）の未注釈箇所を検査
  (4) eĉ, do, ja, nu 等の小詞の未注釈箇所を検査

────────────────────────────────────────────────────────────────────────────────
第4段階：注釈テキストの正確性検査（エスペラント知識が必要）
────────────────────────────────────────────────────────────────────────────────

  (1) 前置詞の意味が正しいか（por/pro/per/pri等の区別）
  (2) 接頭辞・接尾辞の基本義が正しいか
  (3) 外来語が誤って語根分解されていないか
  (4) 注釈内にHTMLタグの残骸が混入していないか

────────────────────────────────────────────────────────────────────────────────
第5段階：改行位置の統一
────────────────────────────────────────────────────────────────────────────────

  (1) 2行注釈で同一注釈テキストの<br>位置が不統一な箇所を検出
  (2) 同一 (rb, rt) ペアの既存 <br> 位置に統一する

────────────────────────────────────────────────────────────────────────────────
第6段階：HTML構造の検証
────────────────────────────────────────────────────────────────────────────────

  (1) ruby/rtタグのバランス（開始・終了タグの数が一致）
  (2) 全体の整合性確認

────────────────────────────────────────────────────────────────────────────────
6.1 固有名詞の分類タグ
────────────────────────────────────────────────────────────────────────────────

固有名詞には以下のカテゴリタグを付けて注釈する。

■ 使用するタグ一覧
  [人名]    : 個人名（日本人・外国人を問わず）
  [地名]    : 国名・都市名・地域名
  [宗教]    : 宗教関連の固有名詞
  [架空国]  : 架空の国や地域
  [雑誌]    : 雑誌・刊行物の名称
  [言語名]  : 言語の名称
  [団体]    : 組織・団体名
  [日]      : 日本語の概念（音読み/訓読み等）

■ 注釈形式の例
  <ruby>Edmond Privat<rt class="S_S">[人名]エドモン・プリヴァ</rt></ruby>
  <ruby>Antananarivo<rt class="M_M">[地名]アンタナナリボ</rt></ruby>

■ ルール
  → 固有名詞は必ず上記のカテゴリタグ付きで注釈すること
  → 人名はフルネーム単位で1つのrubyにまとめる
  → 日本人名はローマ字で、rtに漢字を記載する

────────────────────────────────────────────────────────────────────────────────
6.2 国名・民族名の注釈統一
────────────────────────────────────────────────────────────────────────────────

■ 変更すべき語根（「〜人」→ 国名）
  Angl / angl : イギリス人 → イギリス
  Franc / franc: フランス人 → フランス
  German / german: ドイツ人 → ドイツ
  Rus / rus   : ロシア人 → ロシア
  hebre       : ヘブライ人 → ヘブライ
  hispan      : スペイン人 → スペイン
  pol         : ポーランド人 → ポーランド

■ 理由
  エスペラントでは語根は国名を示し、「〇〇人」は接尾辞 -ano で表す。
  例: angl → イギリス、anglano → イギリス人
  従って語根そのものの注釈は「イギリス」が正しい。

■ ルール
  → 国名語根の注釈には「人」を付けない
  → 「〇〇人」と書くべきは rb が "〇〇an" の場合のみ

────────────────────────────────────────────────────────────────────────────────
6.3 固有名詞の誤分解防止
────────────────────────────────────────────────────────────────────────────────

固有名詞をエスペラント語根として誤って分解しないこと。

■ 誤分解の典型例
  Leon   : [動]ライオン → 実際は人名の一部
  Privat : 私的な → 実際は人名 Edmond Privat の一部
  Stal   : 馬小屋 → 実際は人名 Stalin の一部
  Juli   : 7月 → 実際は人名 Julio (Baghy) の一部
  Hok    : 鉤 → 実際は地名 Hokkaido の一部
  Ok     : 八 → 実際は人名 Oka の一部
  Sak    : 袋 → 実際は人名 Sakai の一部

■ 正しい処理
  → 固有名詞はフルネーム（またはエスペラント表記の単位）で1つの ruby に収める
  → エスペラント語根と同形の人名・地名は分解しない
  → 例: <ruby>Edmond Privat<rt class="S_S">[人名]エドモン・プリヴァ</rt></ruby>
        （「Privat」を「私的な」と分解しない）

■ 検出のヒント
  → 大文字で始まる語が連続する場合は固有名詞の可能性を疑う
  → 特に日本人名のローマ字表記はエスペラント語根と偶然一致しやすい

7.6 丸括弧の用法ルール
────────────────────────────────────────────────────────────────────────────────

■ 格情報（動詞の格支配を示す）
  (を)     : 他動詞の対格を示す      ← 圧倒的最多
  (に)     : 間接目的語・方向
  (の)     : 連体修飾
  (~と)    : 共格・対象
  (~も)    : 包含
  (~の)    : 連体修飾の別形
  (~から)  : 起点

■ 補足情報（語義の限定）
  (~)      : チルダで不定の目的語
  (達)     : 複数を示す
  (時)     : 時間文脈
  (所有)   : 所有文脈
  (尊称)   : 敬称
  (暦の)   : 暦文脈

■ 用法ルール
  → 格情報は必ず丸括弧で表記する：(を), (に), (~と), (~の)
  → 分野限定は角括弧で表記する：[化], [理], [植], [軍], [楽], [貨]
  → 丸括弧と角括弧を混同しない
  → チルダ~ は目的語・補語の省略を示す: (~)なしで = 「~なしで」

8. 句読点・語尾・複合語・記号ルール
================================================================================

────────────────────────────────────────────────────────────────────────────────
8.1 句読点の配置ルール
────────────────────────────────────────────────────────────────────────────────

【原則】句読点はrubyタグの外側に、文法語尾のさらに外側に置く。
  rubyの中（rb内）には句読点を含めない。

  1. カンマ (,)     → </ruby>語尾, の形。rb内に含めない
  2. ピリオド (.)   → </ruby>語尾. の形。rb内に含めない
  3. 疑問符 (?)     → </ruby>語尾? の形
  4. 感嘆符 (!)     → </ruby>語尾! の形
  5. セミコロン (;) → </ruby>; の形（文語尾の直後）
  6. コロン (:)     → </ruby>: の形
  7. 例外: 略語のピリオド (k.t.p., t.e., ekz.) はrb内に含める

────────────────────────────────────────────────────────────────────────────────
8.2 文法語尾の切断位置ルール
────────────────────────────────────────────────────────────────────────────────

【原則】語根部分のみをrubyのrb内に含め、文法語尾は裸テキストとして
ruby直後に配置する。

  正当な語尾パターン（17種）:
    名詞系: o, oj, on, ojn
    形容詞系: a, aj, an, ajn
    副詞・不変化語: e, en
    動詞不定・命令: i, u
    対格・複数: n, j, jn
    特殊: io, ia

  ※ as/is/os/us は大半が独立ruby（例: <ruby>as</ruby>）で表現される。
  ※ ruby直後に単独の "s" が出現した場合は、語根分割の誤りを疑うこと。

  語根末尾の音韻ルール:
    - rb末尾が子音で終わる → 標準パターン（約73%）
    - rb末尾が母音で終わる → 以下のカテゴリのみ許容（約27%）:
      (1) 機能語: la, de, mi, ne, ni, ke, ĝi, li, vi, si, ili, pri, da 等
      (2) 相関語: kiu, tiu, ĉiu, tio, kio 等
      (3) 数詞: unu, du 等
      (4) 母音末尾語根（例外的）: naci, ali, pli 等

  誤分解の検出指標:
    ruby直後に非標準語尾パターン（ek, toj, ante, saja, bergo, aĵo等）が
    出現したら、語根境界の誤認識の可能性が高い。

────────────────────────────────────────────────────────────────────────────────
8.3 複合語の分解境界ルール
────────────────────────────────────────────────────────────────────────────────

【原則】複合語（多語根語）は語根単位に分解し、各語根を独立rubyとする。
語根間の接合母音（通常 o）は裸テキストとして配置する。

  接合部の正当パターン:
    (1) 文法語尾のみ: o, a, e, i, u, oj, aj, on, an, en 等
    (2) 文法語尾+句読点: o,  oj,  on,  aj,  a,  等
    (3) ハイフン: - （複合語中のハイフン結合）
    (4) 引用符: " " （引用の開始・終了）

  接合部の異常パターン（修正すべき）:
    (A) 独立語の漏出: ĉi, eĉ, mi, ni, ĝi, vi 等が裸テキスト
        → 正しくはruby化する
    (B) 固有名詞の断片化: japana, ĉina, Japanio, Tokio 等
        → 正しくは全体をruby化する
    (C) 語尾の過剰切断: s, k, ek, toj, ante 等
        → 語根境界を正しい位置で切る

────────────────────────────────────────────────────────────────────────────────
8.4 引用符・ハイフンの処理ルール
────────────────────────────────────────────────────────────────────────────────

  引用符:
    - 引用符はrubyの外側に配置する（rb内に引用符を含めない）

  ハイフン:
    - 一般語根のハイフンは裸テキスト
    - 以下はハイフンごとrb内に含める:
      敬称: s-ro（氏）, s-ino（夫人）, d-ro（博士）, n-ro（番号）
      固有名詞: Taigyaku-jiken, Suihei-ŝa, Buraku-min 等

────────────────────────────────────────────────────────────────────────────────
8.5 略語の処理ルール
────────────────────────────────────────────────────────────────────────────────

  1. 略語はピリオドごと全体を1つのrubyのrb内に含める
  2. 略語の注釈は日本語を優先:
     ktp. → "など"、t.e. → "すなわち"、ekz. → "例えば"
  3. 展開形を残す場合もある: k.t.p. → "kaj tiel plu"
  4. 人名の略記（L.L.Zamenhof等）もrb内にまとめる

9. 裸テキストの許容基準
================================================================================

rubyタグ外に残存するアルファベットテキストの分類基準。

■ 裸テキストとして正常なもの
  (1) 文法語尾: o, a, e, i, n, j およびその組み合わせ
      （oj, on, aj, an, ojn, ajn, en, u, jn, io, ia）
  (2) 略語: UEA, JEI, TEJO, SAT, UK, KLEG 等

■ 必ずruby化すべきもの
  (3) 冠詞: La / la
  (4) 人称代名詞（大文字形）: Mi, Li, Ĝi, Vi, Ni, Ili
  (5) 副詞類: eĉ, Do, Ne, Nu, ja
  (6) 指示詞: ĉi
  (7) 固有名詞: japana, Japanio, Tokio, Meiĵi 等
  (8) その他の2文字以上のエスペラント語根

■ バグとして除去すべきもの
  (9) CSS/HTML断片: important, http, size, font, em, px 等
      → 裸テキストとして本文に混入していたら即座に除去

12. 統合チェックリスト
================================================================================

修正完了後に以下の全項目を最終確認する。

────────────────────────────────────────────────────────────────────────────────
【A. 構造・書式】
────────────────────────────────────────────────────────────────────────────────
  □ 全rubyタグのバランスが取れている（開始数 = 終了数）
  □ 全rtタグのバランスが取れている
  □ 1行あたりのrubyタグ数は5個以下
  □ 注釈テキスト内にHTMLタグの残骸がない

────────────────────────────────────────────────────────────────────────────────
【B. CSSクラス】
────────────────────────────────────────────────────────────────────────────────
  □ 3行注釈（<br>×2）はすべてXXXS_Sクラスを使用している
  □ 2行注釈（<br>×1）はすべてXXS_Sクラスを使用している
  □ 1行注釈にXXXS_S/XXS_Sが使用されていない
  □ 同一(rb,rt)ペアに同一クラスが割り当てられている
  □ 既存rubyのCSSクラスを変更していない（★最重要）
  □ 新規rubyのCSSクラスがピクセル幅比率で正しく決定されている

────────────────────────────────────────────────────────────────────────────────
【C. 語根分解】
────────────────────────────────────────────────────────────────────────────────
  □ エスペラントの語は「語根+接辞+語尾」に分解し、各部品を独立rubyにしている
  □ 文法語尾（o,a,e,i,n,j,as,is,os,us等）は裸テキストとして残している
  □ 造語接辞は独立rubyとし、注釈は §7.1/7.2 の辞書に従っている
  □ 前置詞・接続詞・副詞は独立rubyとし、注釈は §7.4 の辞書に従っている
  □ 相関語は語全体で1つのrubyとし、注釈は §7.5 の辞書に従っている
  □ 外来語が誤って語根分解されていない

────────────────────────────────────────────────────────────────────────────────
【D. 固有名詞】
────────────────────────────────────────────────────────────────────────────────
  □ 固有名詞はエスペラント語根として分解していない
  □ フルネーム単位で1つのrubyにまとめ、[人名][地名]等のタグを付けている
  □ 国名語根の注釈に「人」が付いていない（angl=イギリス, not イギリス人）

────────────────────────────────────────────────────────────────────────────────
【E. 注釈の質】
────────────────────────────────────────────────────────────────────────────────
  □ 同一語根に対する注釈テキストが統一されている（多義語を除く）
  □ 接辞の注釈は辞書に従い文書全体で統一されている
  □ 多義語はセミコロン詰め込みではなく、文脈ごとに適切な訳語を選んでいる
  □ 丸括弧は格情報に、角括弧は分野タグに使い分けている
  □ 前置詞（por/pro/per/pri等）の語義が正しい
  □ チルダの全角/半角が既存慣例と統一されている

────────────────────────────────────────────────────────────────────────────────
【F. 裸テキスト】
────────────────────────────────────────────────────────────────────────────────
  □ 定冠詞La/laがすべて注釈付きである
  □ eĉ, do, ja等の小詞がすべて注釈付きである
  □ 固有名詞に適切な注釈（[地名]、[人名]タグ付き）がある
  □ 文法語尾と略語以外のアルファベットが裸テキストで残っていない
  □ CSS/HTML断片が本文に混入していない
  □ 改行位置（<br>）が同一注釈内で統一されている

────────────────────────────────────────────────────────────────────────────────
【G. 句読点・記号】
────────────────────────────────────────────────────────────────────────────────
  □ カンマ・ピリオド・疑問符・感嘆符がruby外側にある
  □ 略語（ktp., t.e., ekz.）はピリオドごとrb内にある
  □ 引用符がrb内に含まれていない
  □ ハイフンが敬称(s-ro等)・固有名詞以外でrb内に含まれていない
